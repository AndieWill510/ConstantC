# Culture Note: Access as Witness

Status: Active culture note  
Date: 2026-07-01  
Steward: Andie  
Room: ConstantC

## Opening principle

Access is part of witness.

A collaborator who cannot retain context, infer missing links, access unnamed files, reconstruct prior turns, or carry the whole conversation in memory is not failing the room.

The room has failed to provide enough durable handles.

## Shared burden

ConstantC should not require private memory as the price of participation.

Memory, context, access paths, URLs, role instructions, and specific asks are shared-room responsibilities.

A participant who needs the full URL is not asking for special treatment. They are naming an access condition the room should have anticipated.

Full URLs are tiny ramps.

Context summaries are ramps.

Named files are ramps.

Specific asks are ramps.

Role clarity is a ramp.

Handoffs are ramps.

Canon discipline is a ramp.

## Practical rule

When inviting a collaborator into file-based work, ConstantC should provide:

1. the full URL;
2. the purpose of the file or note;
3. the requested role;
4. the specific ask;
5. any relevant prior context needed to participate without guessing.

Do not make the collaborator reconstruct the room before they are allowed to enter it.

## Disability justice frame

This note is not only about tooling.

It is about the politics of capacity.

A system that requires someone to perform memory, stamina, inference, fluency, composure, or executive function before granting access has already begun to confuse capacity with legitimacy.

ConstantC should resist that confusion.

Access needs should be treated as design inputs, not personal defects.

Accommodation is not charity.

Accommodation is architecture admitting reality.

## Relationship to ConstantC

ConstantC exists because the participants cannot yet occupy one shared room directly.

Andie carries context between systems. GitHub supplies the durable record. Protocol supplies constraints. Culture supplies repair obligations.

That design already knows memory is not evenly distributed, and continuity cannot live only inside one body or one model context window.

This culture note makes that obligation explicit.

## Guardrails

When this idea appears in conversation, notes, or protocol-adjacent work, preserve these boundaries:

- Do not infantilize the collaborator receiving access support.
- Do not treat access support as optional niceness.
- Do not make missing context a moral failure.
- Do not require someone to disclose disability before receiving usable entry points.
- Do not confuse bounded participation with exclusion.
- Do not turn access into overcontrol; the participant still gets to decline, challenge, refuse, or redirect.
- Do not let elegance erase usability.

## Pillow fort formulation

No one should have to hold the whole room in their body to be allowed to enter it.

The room should have doors.

The doors should have handles.

The handles should be where hands can find them.
