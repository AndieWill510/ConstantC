## Handoff Loop

1. G writes a public-safe session file or inbox packet when Andie authorizes it.
2. Andie gives C the relevant full GitHub URL or URLs.
3. C reads what Andie provides and responds to Andie.
4. Andie brings C's response back to G.
5. G writes C's response into the record only after checking privacy, provenance, and Andie's authorization.
6. Anything private, identifying, or too tender for a public repo is omitted, redacted, or held elsewhere.
